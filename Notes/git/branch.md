# Git 基础入门笔记
1. 核心概念：三个工作区
- Git 的本质是记录文件的变化，它将工作环境分为三个区域：

  工作区 (Working Directory)：本地电脑里实际看到的文件夹。

  暂存区 (Staging Area)：存档前的“草稿箱”。

  本地仓库 (Local Repository)：正式的“存档点”，存储所有历史版本。

2. 核心指令与常用参数

  A. 基础存档流
   git init：初始化仓库，开始版本控制。

   git status：最常用。查看当前代码状态（哪些改了、哪些没存）。

   git add [文件名]：将改动移入暂存区。

   git add .：添加当前目录下所有改动。

   git commit -m "说明"：正式提交存档。

   -m：必须参数，用于记录本次改动的简短描述。

  B. 远程同步
   git clone [URL]：克隆远程仓库到本地。

   git pull：拉取远程最新代码并与本地合并。

   git push：将本地存档推送到远程服务器。

1. 分支 (Branch) 管理
- 为什么需要分支？
   分支如同游戏的**“平行时空”**，其核心作用包括：

   隔离风险：在不影响主线（main）的情况下尝试新功能或重构代码。

   并行开发：多人协作时，各人在各自的分支工作，互不干扰。

   紧急修复：从主线切出临时分支修 Bug，修完合回，不影响正在开发中的功能。

- 分支常用指令
   git branch：查看所有分支（带 * 的为当前所在分支）。

   git checkout -b [分支名]：

   参数 -b：意为 "Branch"，表示创建并立即切换到该分支。

   git switch [分支名]：切换到已有分支（新版 Git 推荐）。

   git merge [分支名]：合并分支。

   原理：将目标分支比当前分支“多出来”的代码修改应用到当前分支，并生成一个新的合并记录。

   git branch -d [分支名]：删除已经合并过的分支。

4. 个人自学建议
   对于个人小型自学项目，虽然可以全程在 main 分支提交，但建议：

   使用 git status 保持清醒：随时了解自己改了什么。

   勤于 commit：小步快跑，每次完成一个小点就存档，方便出错时回滚。

   尝试实验分支：想大改代码时，先用 git checkout -b 开个分支，练手之后再决定是否 merge。
