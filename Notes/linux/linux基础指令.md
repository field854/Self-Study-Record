# 基础指令

### 1. 文件与目录操作

这是最常用的指令集，用于导航和管理文件系统。

* **`ls` (List)**：列出目录内容。
* `-l`：显示详细信息（权限、大小、时间）。
* `-a`：显示所有文件（包括以 `.` 开头的隐藏文件）。
* `-h`：以人类易读的格式（如 KB, MB）显示大小。


* **`cd` (Change Directory)**：切换目录。
* `cd ..`：返回上一级。
* `cd ~`：回到用户家目录。


* **`mkdir` (Make Directory)**：创建目录。
* `-p`：递归创建多级目录（如 `a/b/c`）。


* **`rm` (Remove)**：删除文件或目录。
* `-r`：递归删除（用于删除文件夹）。
* `-f`：强制删除，不提示确认。**（慎用！）**


* **`cp` (Copy)**：复制文件或目录。
* `-r`：复制目录及其内容。


* **`mv` (Move)**：移动或重命名文件/目录。

### 2.文件内容查看（重点是“怎么看”）
- tail（查看末尾）

  -f：最重要参数。实时追踪文件更新，常用于监控正在运行的程序日志。

  -n [数字]：指定行数。例如 tail -n 50 查看最后 50 行。

- less（交互式查看）

  -N：在每一行前面显示行号。

  -S：如果一行太长，不进行换行，而是通过左右滚动查看（看数据库导出文件很有用）。

- cat（连接并打印）

  -n：显示行号。

  -A：显示不可见字符（如空格、制表符、换行符），用于排查脚本格式错误。

### 3. 系统信息与权限（重点是“范围与大小”）
- chmod / chown（权限与所属）

  -R：递归（Recursive）处理。这是最常用的，比如将整个文件夹及其内部成千上万个文件的权限一次性全部修改。

- df（磁盘空间）

  -h：人类可读格式。将枯燥的字节数转为 GB、MB 这种一眼能看懂的单位。

- du（文件占用）

  -s：只显示总计，不列出子目录的细节。

  -h：同样是单位转换，通常组合使用为 du -sh。

- ps（进程查看）

  -ef：以标准格式列出所有进程。

  -aux：以更详细的 CPU、内存占用百分比形式列出进程。

- kill（终止进程）

  -9：强制杀掉。不给进程保存数据的时间，直接在内核层面切断运行。

### 4. 文本处理与搜索（重点是“筛选逻辑”）
- grep（文本过滤）

  -i：忽略大小写。搜索 "Error" 时也会匹配到 "error"。

  -v：反向选择。只显示不包含指定字符的行。

  -l：只列出文件名。当你在一堆文件中找包含某个关键词的文件时非常有用。

- find（文件查找）

  -name：按名称查找（支持通配符 *）。

  -type：按类型找。f 代表普通文件，d 代表文件夹。

  -size：按大小找。比如 +100M 找大于 100MB 的文件。

- tar（压缩解压）

  -z：代表使用 gzip 压缩格式。

  -x：代表解压（Extract）。

  -c：代表创建压缩包（Create）。
  
  -v：显示处理过程的进度。

  -f：指定文件名（必须紧跟在文件名之前）。
